Security pillars

Practical controls users can understand

Security should be visible and usable, not hidden in legal text.

Consent-first ingestion

Data streams are opt-in, and users can choose onboarding modes. SMS parsing can be constrained to new events for privacy-forward adoption.

Source provenance

Each transaction can carry metadata about where it came from: manual entry, recurring engine, SMS, or future connector providers.

Isolation by interface

Connector bridge modules sit behind typed adapters, letting infrastructure evolve without exposing raw provider behavior directly in UI.

Progressive hardening

MVP starts with scoped capabilities and gradually adds regulated connectors, retry orchestration, and sync audit metrics.

Connector blueprint

Production path without changing user flows

The same Astra screens can run on manual mode today and switch to approved providers later using connector adapters and token exchange patterns.
